# Web Security Audit – OWASP Top 10

This project demonstrates a security assessment of a web application based on the OWASP Top 10 vulnerabilities.

The objective is to identify security weaknesses in the application and provide recommendations to improve its security posture.

## Project Objectives

- Analyze the attack surface of a web application
- Identify common web vulnerabilities
- Perform vulnerability scanning
- Document security findings

## OWASP Top 10

The audit focuses on common vulnerabilities such as:

- Broken Access Control
- Cryptographic Failures
- Injection (SQL Injection)
- Security Misconfiguration
- Cross-Site Scripting (XSS)

## Tools Used

- Kali Linux
- Burp Suite
- OWASP ZAP
- Nmap

## Audit Methodology

1. Information gathering
2. Attack surface analysis
3. Vulnerability scanning
4. Exploitation testing
5. Security report creation

## Deliverables

The project includes:

- vulnerability analysis
- proof of concept of attacks
- security recommendations

## Skills Demonstrated

This project demonstrates skills in:

- Web application security
- Vulnerability assessment
- Penetration testing methodology
- OWASP security framework
